# Dutch Dictation Flow

Dutch Dictation Flow 是一个面向荷兰语听写训练的纯静态前端应用。它把文章级听写、句子音频、错题本、A2 不规则动词专项、全文默写、学习报告和完整备份集中在一个本地优先的学习工作台中。

项目无需后端服务，适合直接部署到 GitHub Pages，也适合在本地浏览器长期使用。

## 核心能力

- 文章库管理：创建文章、维护来源、标签、难度和句子顺序。
- 逐句听写：每个句子可绑定独立音频，提交后自动评分并记录熟练度。
- 文章归档：完成度 100% 的文章会进入“已学会文章”归档区，并保留定期复习入口。
- 全文默写：已掌握文章可进入全文默写模式，记录全文正确率和用时。
- 错题本：自动或手动记录错词，支持编辑释义、上传单词音频、批量复习和朗读。
- 已掌握错词抽查：归档错词可随机抽查，答错会自动回到待掌握区。
- A2 动词专项：内置 121 个 A2 不规则动词原形，支持朗读、随机复习、指定复习数量、编辑说明和上传音频。
- B1/B2 动词库：自建动词列表，支持添加、编辑、上传音频和拼写复习。
- 学习报告：汇总文章、句子、错词、全文默写、学习时间、掌握率和复习质量。
- 数据备份：支持 JSON 快速备份和 ZIP 完整备份，覆盖文本数据、学习记录、统计快照和音频文件。

## 页面结构

- `文章列表`：文章库首页，区分学习中文章和已学会文章。
- `创建文章`：新建文章并维护基础信息。
- `全文默写`：对已掌握文章进行整篇默写训练。
- `错题本`：集中复习听写过程中暴露的错词。
- `A2动词`：固定 A2 不规则动词专项训练。
- `B1动词` / `B2动词`：自建中高级动词专项训练。
- `报告页面`：查看学习进度、质量趋势和复习规划。
- `数据备份`：导出或导入学习数据。

## 复习与评分

听写评分基于文本归一化和编辑距离计算。应用会记录每次提交的答案、分数、正确文本和时间戳，并据此更新熟练度。

已支持荷兰语中常见的撇号写法，例如：

- `'s avonds`
- `m'n`
- `vooral's`

不同样式的撇号会被统一处理，但不会被删除，因此听写比对可以识别这类拼写差异。

## 数据存储

应用采用本地优先架构：

- `localStorage`：保存文章、句子文本、标签、复习历史、错题本、A2 动词进度、全文默写记录和统计数据。
- `IndexedDB`：保存句子音频、错题音频和 A2 动词音频。

这些数据会在刷新或关闭浏览器后保留。若清除浏览器数据、更换浏览器或更换设备，本地数据可能丢失，因此建议定期导出备份。

## 备份格式

备份文件包含可恢复数据和统计快照两类内容。

主要可恢复字段：

- `articles`：文章、句子、句子音频引用、复习记录和全文默写记录。
- `wordBook`：错题本词条、释义、音频引用、熟练度和历史记录。
- `customVerbDecks`：B1/B2 自建动词库，包含动词、中文说明、音频引用、熟练度和复习历史。
- `customVerbDeckReviewSettings`：B1/B2 自建动词库复习格式，包括全部复习、未掌握复习和最陌生复习。
- `wordReviewSessions`：单词复习 session。通过 `deck` 区分错题本与 A2 动词，通过 `mode` / `modeLabel` / `random` / `weakest` / `limit` 记录本次复习方式。
- `a2Vocabulary`：内置 A2 不规则动词词表元信息。
- `a2VocabularyReviewSettings`：A2 动词批量复习格式，包括全部复习、未掌握复习、最陌生复习、随机复习、随机未掌握和随机最陌生。
- `a2VocabProgress`：A2 动词个人学习进度、编辑后的说明、音频引用和复习历史。
- `audioFiles`：音频文件。JSON 快速备份中以内联数据保存；ZIP 完整备份中以独立文件保存。

主要统计快照：

- `articleLibraryStats`：文章库归档统计，包括学习中文章、已学会文章、已掌握句子和复习轮数。
- `wordBookMasteryStats`：错题本掌握统计，对应“完全掌握 / 基本掌握 / 需要复习 / 未开始”。
- `customVerbDeckMasteryStats`：B1/B2 自建动词库掌握统计。
- `a2VocabularyMasteryStats`：A2 动词掌握统计。
- `reportData`：新版报告仪表盘快照，包含学习时间、复习质量、日历、行动建议、剩余规划、错题本状态、全文默写状态和 A2 动词专项状态。

导入时以原始可恢复数据为准，统计快照用于展示、校验或外部分析。导入后应用会根据恢复后的数据重新计算页面统计。

## 本地运行

项目是纯静态应用，可以直接打开 `index.html`。为了获得更稳定的浏览器行为，推荐用本地静态服务器运行：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 部署

可直接部署到 GitHub Pages：

1. 将仓库推送到 GitHub。
2. 在仓库设置中启用 GitHub Pages。
3. 选择主分支和根目录作为发布源。
4. 打开生成的 Pages 地址。

项目没有构建步骤，也没有后端依赖。

## 目录说明

```text
.
├── index.html              # 应用页面结构
├── app.js                  # 核心交互、评分、存储、导入导出逻辑
├── styles.css              # 页面样式与响应式布局
├── favicon.svg             # 浏览器图标
├── dutch-listen-icon.svg   # 应用图标
└── audio/                  # 可选音频资源目录
```

## 使用建议

- 每篇文章按句子拆分，并尽量为每句上传独立音频。
- 先通过逐句听写把文章练到 100%，再进入全文默写。
- 错题本中已掌握的词建议定期随机抽查。
- A2 动词专项适合每天随机抽取 10-20 个做拼写复习。
- 每次集中学习后导出一次备份，尤其是在更换设备或清理浏览器缓存前。

## 设计原则

- 本地优先：学习数据默认保存在使用者自己的浏览器中。
- 无后端依赖：部署和迁移成本低，适合个人长期维护。
- 可恢复：文本数据、音频引用、复习历史和统计快照都进入备份格式。
- 面向复习闭环：听写、发现问题、沉淀错词、归档掌握、定期抽查形成完整学习流程。
